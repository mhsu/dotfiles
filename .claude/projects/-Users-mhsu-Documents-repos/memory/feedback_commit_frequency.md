---
name: commit_frequency
description: When to commit in the aiagent project
type: feedback
---

Don't commit after every single small change. Batch related changes and only commit when:
- The user explicitly asks to commit/push
- A meaningful feature or fix is fully complete

**Why:** User found per-change commits noisy and interrupted their flow.
**How to apply:** Make all the edits, then wait for "commit and push" or natural stopping point.
