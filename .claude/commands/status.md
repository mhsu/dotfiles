# Project status

The user wants to know where things stand on the current project.

Follow these steps:

1. **Read the project context.**
   - Read context.md in the current folder if it exists
   - Run git log --oneline -5 to see recent commits
   - Run git status to see any uncommitted changes
   - List the files in the current project folder (not subfolders)

2. **Give the user a clear status summary in this format:**

   **What this project is:** [1 sentence]

   **Current phase / status:** [e.g. "Phase 2 done, Phase 3 not started"]

   **What works right now:** [what can the user actually do or see today?]

   **What was last worked on:** [based on recent commits or context.md]

   **What's next:** [top 2-3 priorities from context.md or your assessment]

   **Any loose ends:** [uncommitted changes, known bugs, things to watch out for]

3. **Keep it short.** No more than 15 lines total. Plain English only.
   No technical jargon. If something needs explaining, translate it.
